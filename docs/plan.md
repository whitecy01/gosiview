# 고시원 관리 프로그램 요구사항 명세서

## 문서 개요

- **문서명**: 고시원 관리 프로그램 요구사항 명세서
- **작성일**: 2026년 3월 26일
- **작성자**: 정재윤
- **현재 버전**: v0.2
- **프로젝트 목적**: 고시원의 입실 현황, 장부, 연간 캘린더, 방 상세 정보 등을 관리할 수 있는 웹 기반 운영 도구를 개발하기 위한 요구사항을 정리한다.

---

## 1. 프로젝트 배경

본 프로젝트는 고시원의 운영 상태를 효율적으로 관리하기 위한 **웹 기반 프로그램**을 만드는 것을 목표로 한다.

현재 운영자는 **사장님, 원장님, 총무님**으로 구성되어 있으며, 주요 관리 대상은 다음과 같다.

- 현재 어느 방에 누가 거주 중인지
- 입실 시작일 / 종료일
- 월세 납부 여부 및 납부 이력
- 공실 여부
- 연간 입실 현황
- 방 상세 정보 및 유지보수 정보

또한 본 프로그램은 고정된 PC가 아니라 **태블릿 중심 사용 환경**을 우선 고려한다.

---

## 2. 사용자 정의

각 사용자는 프로그램을 사용하는 목적과 관점이 다르므로, 역할별 주요 사용 목적을 정의한다.

### 2.1 사장님

- 전체 운영 현황 확인
- 연간 캘린더 확인
- 방 상세 정보 확인
- 장부 리스트 관리
- 정산, 미납, 승계 정보 확인
- 신규 입실자 등록
- 방 상태 및 상세 정보 확인

### 2.2 원장님

- 전체 운영 현황 확인
- 도면 기반 입실 현황 확인

### 2.3 총무님

- 방 정보 입력
- 비품 / 소모품 / 유지보수 정보 관리
- 입실자 관련 기초 데이터 관리

## 4. 화면별 요구사항

### 4.1 대시보드(도면 기반)

#### 목적

고시원 전체 방 상태를 한눈에 확인한다.

#### 주요 기능

##### 4.1.1 1층~6층 도면 표시
- 사용자는 고시원의 형태를 1층부터 6층까지 도면 형태로 확인할 수 있다.

##### 4.1.2 공실 / 입실중 / 계약상태 표시
- 사용자는 도면에서 방 상태인 공실 / 입실 중 / 계약상태를 확인할 수 있다.

##### 4.1.3 방 클릭 시 상세 정보 표시
- 사용자는 도면에서 방을 클릭했을 때 상세 정보를 볼 수 있다.
- 사용자는 입실자 관리에 등록된 데이터를 상세 정보에서 확인할 수 있다.

##### 4.1.4 신규 입실자 등록 연결
- 사용자는 공실 상태의 방을 클릭한 뒤 상세 정보에서 **신규 입실자 등록** 버튼을 통해 입실자를 등록할 수 있다.

##### 4.1.5 확대 / 축소 기능
- 사용자는 도면 및 방의 상세 정보를 확대 / 축소하여 볼 수 있다.

##### 4.1.6 수치 데이터
- 사용자는 총 방 개수, 총 입실자 수, 현재 공실, 유지보수 현황을 확인할 수 있다.
- 각 수치 카드를 클릭하면 해당 데이터에 포함된 목록을 확인할 수 있다.

#### 논의 사항

- 없음

#### 개발 상태

- **4.1.1 1층~6층 도면 표시** → 일부 구현 완료
  - 현재는 그린 형태이며, 실제 도면이 들어가는 형태로 변경 필요
- **4.1.2 공실 / 입실중 / 계약상태 표시** → 일부 구현 완료
  - 유지보수를 계약 상태와 구분해서 정리 필요
- **4.1.3 방 클릭 시 상세 정보 표시** → 일부 논의중
  - 입실자 관리 데이터 전체 연동 필요
- **4.1.4 신규 입실자 등록 연결** → 구현 완료
- **4.1.5 확대 / 축소 기능** → 구현 필요
- **4.1.6 수치 데이터** → 일부 구현 완료
  - 카드 클릭 시 상세 목록 확인 기능 필요

### 4.2 입실자 관리

#### 목적

입실자와 월세 납부 상태를 표 형태로 관리한다.

#### 주요 기능

##### 4.2.1 입실자 목록 조회
- 사용자는 표 형태에서 **호실, 이름, 연락처, 상세 보기 버튼**을 볼 수 있다.
- 사용자는 각 컬럼을 구분된 색상으로 볼 수 있다.
- 사용자는 **예정 입실 정보**를 확인할 수 있다.
  - 예: 오늘이 3월 24일인데 26일부터 살 사람이 이미 정해져 있고, 다른 사람이 27일부터 오겠다고 문의하는 상황
  - 단순 공실 여부가 아니라 **미래 예정 입실 일정**까지 보여줄 수 있어야 한다.
- 사용자는 **방 관리 버튼**을 통해 다음 정보를 확인할 수 있다.
  - 날짜 (`26/03/24`)
  - 금액
  - 방 관리 상세
    - 도배
    - 매트리스 교체
    - 에어컨 청소
    - 전구 교체

##### 4.2.2 입실자 상세 조회
- 사용자는 입실자 목록의 **상세 보기 버튼**을 클릭하여 다음 데이터를 확인할 수 있다.

###### 기본 정보
- 호실
- 입실자 이름
- 연락처
- 호실(방 유형)
  - `Cozy`
  - `Standard A-1`
  - `Standard A-1 +`
  - `Standard A-2`
  - `Standard A-2 +`
  - `Standard A-2 (넓은 사이즈)`
  - `Standard A-3`
  - `Standard B-1`
  - `Standard B-2`
  - `Deluxe A`
  - `Deluxe B`
- 금액(관포)
- 입실일
- 퇴실일
- 계약 만료일
- 거주 목적
  - 공시생(임용, 일행, 소방, 경찰 등)
  - 세무
  - 회계
  - 계리
  - 취준생
  - 수능
  - 직장
- 실제 납부해야 할 월세
- 납부 대상 월일
- 계약금
- 부동산 정보

###### 보증금 관리 이력
- 잔여 보증금 내역
- 차감 이행 날짜
- 차감 보증금 금액
- 차감 사유
  - 차임
  - 미납
  - 공과금 정산
  - 도배
  - 타일
  - 시설 손상

###### 월세 납부 관리 및 내역
- 납부한 날짜
- 납부한 금액
- 결제 방식
  - 이체(자진 발급)
  - 이체
  - 현금
- 월별 납부 내역
  - 입실 시점부터 현재까지 월 단위 리스트 형태
- 납부 상태
  - 미납
  - 납부 완료
- 자동 처리 예시
  - 매월 3일 기준
  - 3일이 지나도록 값이 없으면 미납 처리
  - 해당 월에 값이 입력되면 자동 납부 완료 처리

###### 공과금 처리
- 도시가스
  - 월별 납부 내역 확인
  - 직접 등록 / 수정 가능
- 한전
  - 현금 승계 처리 필요
  - 퇴실일과 실제 검침일 사이 사용 금액 입력 가능
  - 다음 입실자에게는 해당 금액만큼 제외된 금액이 반영되어야 함

##### 4.2.3 수치 데이터
- 사용자는 총 방 개수, 총 관리 입실자, 이번 달 납부 완료, 확인 필요한 미납 수치를 볼 수 있다.
- 각 수치 카드를 클릭하면 포함된 입실자 목록을 확인할 수 있다.

#### 논의 사항

- 없음

#### 개발 상태

- **4.2.1 입실자 목록 조회** → 일부 구현 완료
  - 컬럼 색상 구분 기능 필요
  - 예정 입실 정보 기능 필요
- **4.2.2 입실자 상세 조회** → 구현 필요
  - 전반적인 상세 기능 개발 필요
- **4.2.3 수치 데이터** → 일부 구현 완료
  - 카드 클릭 시 상세 목록 확인 기능 필요

### 4.3 연간 캘린더

#### 목적

시간 축 기준으로 각 호실의 입실 현황을 시각적으로 확인한다.

#### 주요 기능

##### 4.3.1 호실별 막대 그래프 표시
- 사용자는 막대 그래프 위에서 다음 정보를 확인할 수 있다.
  - 입실자 이름
  - 호실
  - 월세 값
  - 입실일
  - 퇴실일

##### 4.3.2 막대 그래프 상세 정보
- 사용자는 하나의 막대 그래프를 클릭했을 때 해당 데이터의 상세 정보를 볼 수 있다.
- 상세 정보에는 입실자 관리에 해당하는 모든 데이터를 포함한다.

##### 4.3.3 층별 필터링
- 사용자는 층별로 필터링하여 캘린더를 확인할 수 있다.

#### 논의 사항

- 없음

#### 개발 상태

- **4.3.1 호실별 막대 그래프 표시** → 일부 구현 완료
  - 컬럼 색상 구분 필요
  - 월세 값을 `70`, `75`처럼 축약해서 보이는 형태 필요
  - 모든 연도를 볼 수 있는 스크롤 형태 검토 필요
- **4.3.2 막대 그래프 상세 정보** → 구현 필요
  - 입실자 관리 상세 데이터 전체 연동 필요
- **4.3.3 층별 필터링** → 구현 완료

### 4.4 Todo List (Main Page)

#### 목적

총무님들이 해야 할 일을 원장님과 사장님과 공유한다.

##### 4.4.1 Todo List 조회
- 사용자는 월별 캘린더 형태로 조회할 수 있다.
- 캘린더 안에서 해야 할 일을 확인할 수 있다.

##### 4.4.2 Todo List 입력
- 사용자는 Todo List를 직접 입력할 수 있다.

#### 논의 사항

- 없음

#### 개발 상태

- **4.4.1 Todo List 조회** → 구현 필요
- **4.4.2 Todo List 입력** → 구현 필요
